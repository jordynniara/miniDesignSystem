Design System: Components - Cards

Cards

A container to summarize content typically with an image, title, short description, and a button to open a larger container like a modal. Button can be omitted.

Accessbility Considerations:

Coloring for the cards are chosen to fit the theme of the system while also maintaining WCAG color contrast standards. Links should not be styled as buttons.Cards should have text, be focusable, and have the appropriate aria labelling for keyboard navigation and screen readers. See Mozilla Web Documents for more accessibility guidelines.

Default Card

Primary card to be used to preview projects with a picture, title, and short description. Button can added to open a larger description in a modal or similar pop-up, or can be swapped with a link to navigate to an an external site such as GitHub.
Code shown is for card on display. CSS can be modified as needed using classes .card, .card-body, .card-header and .card-text
