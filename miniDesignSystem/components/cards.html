Design System: Components - Cards

Cards

A container to summarize content typically with an image, title, short description, and a button to open a larger container like a modal. Button can be omitted or swapped with a link.

Default Card

Title

Lorem ipsum dolor sit amet, consectetuer adipiscing elit. Aenean commodo ligula eget dolor. Aenean massa. Cum sociis natoque penatibus et magnis dis parturient montes, nascetur ridiculus mus. Donec quam felis, ultricies nec, pellentesque eu, pretium quis, sem. Nulla consequat massa quis enim.

Primary card to be used to preview projects with a picture, title, and short description. Button can added to open a larger description in a modal or similar pop-up, or can be swapped with a link to navigate to an an internal or external site such as GitHub.
Overflow is turned off to keep body text short but can be added with Overflow-y:scroll.
Code shown is for card on display. CSS can be modified as needed using classes .card, .card-body, .card-header, .card-text, and .card-footer

<div class="card">
 <img src="../assets/images/dragon.jpg" class="card-img-top" alt="...">
 <div class="card-body">
  <div class="card-header">
   <h4 class="m-0">Title</h4>
  </div>
  <p class="card-text">
Lorem ipsum dolor sit amet, consectetuer adipiscing elit. Aenean commodo ligula eget dolor. Aenean massa. Cum sociis natoque penatibus et magnis dis parturient montes, nascetur ridiculus mus. Donec quam felis, ultricies nec, pellentesque eu, pretium quis, sem. Nulla consequat massa quis enim.
  </p>
  <div class="card-footer ">
   <button type="button" class="btn primary-sky">Button</button>
  </div>  </div>
</div>

Coloring for the cards are chosen to fit the theme of the system while also maintaining WCAG color contrast standards. Links and Buttons should be used distinctly from each as defined in Buttons. Cards should have text, be focusable, and have the appropriate aria labelling for keyboard navigation and screen readers. See Mozilla Web Documents for more accessibility guidelines.

An expanded version of the card that can be used similarly to the default card but when more text is required. Button can added to open a larger description in a modal or similar pop-up, or can be swapped with a link to navigate to an an internal or external site such as GitHub. Has a wide layout for larger screens and a narrow layout for smaller screens (reduce screen size for demo).
