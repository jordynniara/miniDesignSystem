Design System: Components - Buttons

Buttons

Three states, two styles, three variations.
Used to complete actions on a page or within other components.

Primary - Active

To trigger a primary action. CSS can be modified as needed using classes .btn or .btn-* where * is one of strawberry, sky, or inchworm for each color variation

<button type="button" class="btn primary-strawberry">Button </button>
<button type="button" class="btn primary-sky">Button </button>
<button type="button" class="btn primary-inchworm">Button </button>

Coloring for the buttons are chosen to fit the theme of the system while also maintaining WCAG color contrast standards. Links should not be styled as buttons. Buttons should maintain a minimum target size of 44 x 44 pixles. Buttons should have text, be focusable, and have the appropriate aria labelling for keyboard navigation and screen readers. See Mozilla Web Documents for more accessibility guidelines.

Secondary - Active

To trigger a secondary action. Should only be used when a primary actions is present. .btn or .btn.secondary-* where * is one of strawberry, sky, or inchworm for each color variation

<button type="button" class="btn secondary-strawberry">Button </button>
<button type="button" class="btn secondary-sky">Button </button>
<button type="button" class="btn secondary-inchworm">Button </button>

See Mozilla Web Documents for more accessibility guidelines.
