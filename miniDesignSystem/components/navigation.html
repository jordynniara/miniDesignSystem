Default responsive navigation bar. Can utilizes Bootstrap's Offcanvas class to toggle visibility. See documentation for details and this site's vertical navigation and menu toggle above for demonstration.
Code shown is for card on display. CSS can be modified as needed using classes .navbar-expanded, .nav-header, .nav-list, , and .nav-footer.

<nav class="navbar-expanded">
 <div class="nav-header">
  <a href="#">
   <img class="logo" src="../assets/logos/reverse - color.png" alt="JNS Logo"></img>
  </a>
 </div>
 <ul class="nav-list">
  <li class="mb-1 red-hover">
  <button class="btn btn-toggle d-inline-flex align-items-center rounded border-0 collapsed" data-bs-toggle="collapse" data-bs-target="#demo-collapse" aria-expanded="false">
Expandable Item
  </button>
  <div class="collapse" id="demo-collapse">
   <ul class="btn-toggle-nav list-unstyled fw-normal pb-1 small">
     <li><a href="#" class="d-inline-flex rounded">Item A</a></li>
    <li><a href="#" class="d-inline-flex rounded">Item B</a></li>
    <li><a href="#" class="d-inline-flex rounded">Item C</a></li>
    </ul>
   </div>
  </li>
  <li class="blue-hover"><a class="d-inline-flex rounded" href="#">Item 2</a></li>
  <li class="green-hover"><a class="d-inline-flex rounded" href="#">Item 3</a></li>
  </ul>
 <div class="nav-footer border-top" style="color:white;">
Footer
 </div>
</nav>

Coloring for the navigation are chosen to fit the theme of the system while also maintaining WCAG color contrast standards. Links and Buttons should be used distinctly from each as defined in Buttons. Items in navigation bar should be focusable, and have the appropriate aria labelling for keyboard navigation and screen readers. See Mozilla Web Documents for more accessibility guidelines.
